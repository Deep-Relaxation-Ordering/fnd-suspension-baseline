# Release notes — `pilot-v0.4`

*Endorsement Marker: Local stewardship — U. Warring, AG Schätz, Physikalisches
Institut Freiburg.*

`pilot-v0.4` is a **two-housekeeping-slice** release on top of
`pilot-v0.3`. It closes two L1 slices from
[`docs/program-context.md` §3.1](../program-context.md) (S3 —
Hydrodynamic-shell calibration per FND class; S5 — Concentration-
weighted polydispersity kernel), tightens the parallel-walk
infrastructure for macOS, adds a parameter-sweep root-finder, and
sweeps doc-fix and review residue from the v0.3 cycle. It stays
anchored to the same breakout-note v0.2 spec (commit `3b7b18af`) and
preserves the v0.3 §5.1 regime-label surface by zero-default
compatibility paths.

## What changed from `pilot-v0.3`

### B. S3 — Hydrodynamic-shell calibration per FND class

[`src/parameters.py`](../../src/parameters.py) adds a
`DeltaShellCalibration` dataclass, four canonical class defaults
(`bare = 0 nm`, `carboxylated = 5 nm`, `hydroxylated = 0 nm`,
`peg_functionalised = 7 nm`), `FND_CLASS_ALIASES`, the helpers
`delta_shell_for_fnd_class(...)` and
`delta_shell_calibration_for_fnd_class(...)`, and the new
`ParticleGeometry.from_fnd_class(...)` opt-in constructor.
[`docs/delta_shell_calibration.md`](../delta_shell_calibration.md)
is rewritten as a citation-anchored calibration surface with a
source register. The v0.3 audit-gap pin around generic planning
defaults is retired; campaign-batch DLS remains the v1.0-grade
measurement.

`ParticleGeometry(...)` and `ParticleGeometry.from_radius(...)`
remain at `delta_shell_m = 0` for byte-identical v0.3 reproduction.

### E. S5 — Concentration-weighted polydispersity kernel

[`src/polydispersity.py`](../../src/polydispersity.py) adds a
`weighting: Literal["classification", "number_density"]` keyword
to `lognormal_smear(...)`. Default `"classification"` reproduces
v0.3's marginal channels byte-identically. The new
`"number_density"` mode populates two additional `(3, ...)` arrays
on `SmearedGrid` — `expected_radius_by_regime` and
`expected_radius_sq_by_regime` — that carry per-regime conditional
radius moments. The first axis indexes regime in `(H, S, SED)` order
via the new `REGIME_INDEX_HOMOGENEOUS / _STRATIFIED / _SEDIMENTED`
constants. A tracking experiment can now ask "given an observed
regime label, what is the expected radius and radius²?" without
re-walking the cache.

### I. Spawn-context for `walk_grid`

[`src/regime_map.py`](../../src/regime_map.py) switches the
`ProcessPoolExecutor` start method to
`multiprocessing.get_context("spawn")`. The serial path
(`n_workers = 1`) is unchanged. The macOS fork-safety caveat from
v0.3 release notes §H is removed: pre-fork imports of NumPy /
SciPy under macOS' Accelerate framework no longer deadlock at
worker creation.

A user-facing guard now raises a clear `RuntimeError` when
`n_workers > 1` is invoked from stdin or a non-importable
pseudo-main (for example a heredoc / `python - <<'PY'` session),
instead of failing later with `BrokenProcessPool`. The fix is to
call from a `.py` file / `python -m` module / importable `python -c`
entrypoint, or to stay on `n_workers = 1`.

### J. `crossing_parameter` parameter-sweep root-finder

[`src/time_evolution.py`](../../src/time_evolution.py) adds
`crossing_parameter(...)`, the parameter-sweep counterpart to v0.3's
`crossing_time`. Given a fixed cell and observation time, it
sweeps `parameter ∈ {"delta_shell_m", "lambda_se"}` over an
inclusive `[p_min, p_max]` interval, brackets a sign-change in
`bmf` or `top_to_bottom_ratio`, and refines the crossing with PCHIP
+ Brent's method (linear-interpolation fallback). Returns `None`
when no crossing exists in the interval.

The v0.3 ratio-criterion crossings (`crossing_time(criterion='ratio')`)
were already shipped in v0.3 Phase 22; v0.4's contribution is the
parameter-sweep extension.

## Forward-compatibility contract

All zero-default paths (`lambda_se = 1.0`, `delta_shell_m = 0`,
`delta_T_assumed = 0.0 K`, `n_workers = 1`,
`weighting = "classification"`) reproduce the v0.3 §5 cache to
machine precision. The integration audit (Phase 31) verified this
on a 24-cell focused subset against the committed 6300-cell cache,
plus dedicated smoke tests for the Phase 27 / 28 / 30 surfaces.

The `provisional = True` API contract from
[ADR 0002 §"API surface"](../adr/0002-v0.3-spec-anchoring.md#api-surface--the-provisionaltrue-contract)
remains in force throughout v0.4 because S1 has not landed; it
will clear when S1 (DLVO aggregation pre-screen) lands in a future
cycle.

## DOI status

No DOI is embedded, and none is planned for the v0.4.x series. A
Zenodo DOI remains deferred to the `pilot-v1.0` release: pre-v1.0
pilots have a moving physics scope, and a citation-grade identifier
should not be minted against a deliberately unstable surface. Until
v1.0, cite this repository by its GitHub URL and tag
(`pilot-v0.4`). See the Phase 16.1 lab note for the deferral
decision.

Candidate tightenings for the next pilot slice, drawn from the
[`program-context.md` §3.1](../program-context.md) S1–S7 menu plus
the v0.4 work plan §1 deferrals:

  Promotes to first-slice candidate once the upstream
  `Deep-Relaxation-Ordering/diamonds_in_water` v0.3 breakout note
  lands. Closing S1 also retires the `provisional = True` API
  contract.

  defining; sibling-repo breakout note required.
- **S7 — Thermal control as first-class axis.** Gated on D-PC-1
  (campaign protocol existence).
- **`lambda_se` → §5 scan axis** (work-plan-v0-4 item C, conditional
  defer). Conditional on the S2 calibration in the FND band
  showing working values ≤ 0.3.
- **Concentration-weighted design-table notebook** (work-plan-v0-4
  Phase 28 follow-up). Promote the `weighting="number_density"`
  output into a new design table or notebook surface for downstream
  tracking-experiment consumers.
- **`crossing_parameter` design table** (work-plan-v0-4 Phase 30
  follow-up). Same shape — promote the kernel into a published
  design table once a campaign provides target metrics.

None of these are v0.4 correctness fixes; they are the next layer
of L1 closure toward `pilot-v1.0`.
