# Release notes — `pilot-v0.3`

*Endorsement Marker: Local stewardship — U. Warring, AG Schätz, Physikalisches
Institut Freiburg.*

`pilot-v0.3` is the **implementation-tightening** release on top of
`pilot-v0.2`. It stays anchored to the same breakout-note v0.2 spec
(commit `3b7b18af`) and preserves the v0.2 §5.1 regime-label surface
by zero-default compatibility paths, while adding continuous channels,
a fidelity envelope, and a first Stokes–Einstein correction slice
that were flagged as audit-gap items during v0.2 closeout.

## What changed from `pilot-v0.2`

### K. Stokes–Einstein corrections at sub-150-nm radii (S2)

[`src/stokes_einstein_correction.py`](../../src/stokes_einstein_correction.py)
adds a `lambda_se` breakdown coefficient to the diffusivity and
regime-map channels. At `lambda_se = 1.0` (default) the v0.2
byte-identical surface is reproduced; at `lambda_se < 1.0` the
sub-150-nm diffusivity is reduced, producing provisional regime-label
flips that are flagged with a `provisional = True` marker.

### A. Audit-gap pins resolved — `T_OBS_S` and `DEPTHS_M`

The six observation times and the 10 mm cuvette depth are formally
named as v0.3 defaults under ADR 0002 D1 (no numerical change; the
values already in `scan_grid` are the authority). Pin resolved with
cross-reference documentation only.

### B. Continuous regime thresholds

[`src/continuous_thresholds.py`](../../src/continuous_thresholds.py)
replaces grid-snapped design-table entries with root-finding
(`scipy.optimize.brentq`) on `top_to_bottom_ratio = 0.95` and
`bottom_mass_fraction = 0.95`. New room-T continuous CSVs ship
alongside the unchanged grid-snapped tables; the §5 cache is
untouched.

### C. Mesh-convergence fidelity envelope

Notebook `07_mesh_convergence_audit.py` documents a formal sweep:
bottom-mass drift < 0.006 % at 30 cells; ratio drift < 0.4 % at 240
cells. The existing 120/240-cell first-pass / refinement strategy is
confirmed and now has an audit trail.

### F. `delta_shell_m` provisional calibration table

[`docs/delta_shell_calibration.md`](../delta_shell_calibration.md)
collects literature estimates for functionalised nanodiamond shell
thicknesses. The values are marked provisional and pinned as an
audit-gap item pending a campaign DLS measurement.

### H. Parallel `walk_grid`

[`src/regime_map.py`](../../src/regime_map.py) gains a `n_workers`
kwarg on `walk_grid()`. The default `n_workers = 1` is serial and
byte-identical to v0.2. `n_workers > 1` uses
`ProcessPoolExecutor`; macOS users should note the fork-safety
caveat (pre-fork imports can hang — use spawn context or stay on
serial default).

### J. Continuous time-evolution channel

[`src/time_evolution.py`](../../src/time_evolution.py) adds
`time_series()` (evaluate a cell at arbitrary `t_obs`) and
`crossing_time()` (two-stage log-bracket + PCHIP + Brent's method
refinement to find when a metric crosses a target). Notebook
`08_time_evolution_design_table.py` generates crossing-time design
tables.

## Forward-compatibility contract

All zero-default paths (`lambda_se = 1.0`, `delta_shell_m = 0`,
`delta_T_assumed = 0.0 K`, `n_workers = 1`) reproduce the v0.2 §5
cache to machine precision. The integration audit (Phase 23) verified
this on a 24-cell focused subset against the committed 6300-cell
cache.

## DOI status

No DOI is embedded, and none is planned for the v0.3.x series. A
Zenodo DOI remains deferred to the `pilot-v1.0` release: pre-v1.0
pilots have a moving physics scope, and a citation-grade identifier
should not be minted against a deliberately unstable surface. Until
v1.0, cite this repository by its GitHub URL and tag
(`pilot-v0.3`). See the Phase 16.1 lab note for the deferral
decision.

Candidate tightenings for the next pilot slice, drawn from the
[`program-context.md` §3.1](../program-context.md) S1–S7 menu:

- **S1 — DLVO aggregation pre-screen** (cell trustworthiness flag).
  Gated on the upstream `Deep-Relaxation-Ordering/diamonds_in_water`
  v0.3 breakout note.
- **S3 — Hydrodynamic-shell calibration per FND class.** Replace
  `delta_shell_m` from its v0.3 user-supplied-knob status with a
  citation-anchored default per FND class.
- **S4 — Capsule-geometry port (1-D radial in spherical coordinates).**
  Sealed micro-cells at d = 10–100 µm; sibling-repo breakout note
  required first.
- **S5 — Concentration-weighted polydispersity kernel.** Bounded
  change to `lognormal_smear`'s weighting kernel.
- **S6 — Wall-hydrodynamic Faxén/Brenner corrections.** Position-
  dependent corrections to drag and diffusivity near sealed-cell
  boundaries.
- **S7 — Thermal control as a first-class axis.** Promote
  `delta_T_assumed` from side-channel to first-class axis;
  sealed-cell only.

Plus tactical follow-ups outside the S-slice menu:

- Promote `lambda_se` to a §5 scan axis if calibrated working
  values in the FND band are ≤ 0.3.

None of these are v0.3 correctness fixes; they are the next layer of
experimental realism and consumer convenience. The v0.4 work plan
([`docs/work-plan-v0-4.md`](../work-plan-v0-4.md), Phase 26) records
which subset v0.4 actually adopts.
