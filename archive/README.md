Per
[cd-rules §0.8](https://github.com/threehouse-plus-ec/cd-rules/blob/main/blueprint-threehouse-CD.md)
("deprecation, not deletion"), superseded files are moved here rather than
removed. Git history is not a substitute for visible archival.

## Naming convention (cd-rules §15 deprecation protocol)

Move the deprecated file here and add a deprecation note alongside it:

```
archive/YYYY-MM-DD-<original-filename>-deprecated.md
```

The deprecation note should record: what the file was, why it is being
retired, and what (if anything) supersedes it. Tag the repo
(`pilot-vX.Y.Z`) at the time of deprecation so the pre-archive state is
recoverable from the tag.
